# Checklist da Atividade - Guia Turistico Digital

## Requisitos obrigatorios

- [x] Navegacao principal com TabBar para Pontos Turisticos e Restaurantes
- [x] Drawer lateral envolvendo as Tabs
- [x] Lista de itens com nome, imagem e pequena descricao
- [x] Tela de detalhes com imagem maior, nome, descricao completa e localizacao
- [x] Passagem de parametros via navegacao com `route.params`
- [x] Consumo de dados via JSON local e camada preparada para API
- [x] Uso de biblioteca de UI com cards, botoes e componentes estilizados
- [x] Interface organizada com estrutura separada em `src/screens/`
- [x] Prototipo visual alinhado ao estilo editorial do Figma

## Ajustes feitos para o mobile

- [x] Layout responsivo com rolagem vertical nas telas principais
- [x] Faixa com rolagem lateral para destaque no mobile
- [x] Cards, headers e blocos com espacamento maior em telas pequenas
- [x] Telas mais legiveis no Expo Go ao escanear o QR code

## Dados e conteudo

- [x] 30 itens disponiveis em Pontos Turisticos na base local
- [x] Imagens curadas por categoria para reduzir repeticao e fotos aleatorias
- [x] Fallbacks preparados caso a API nao responda

## Extras implementados

- [x] Tela de login/editorial extra
- [x] Botao de favoritar na tela de detalhes
- [x] Terceiro icone extra na navegação inferior como detalhe visual
- [x] Melhorias visuais drasticas em Sobre e Contato

## Observacao

- O app esta preparado para funcionar mesmo sem rede, priorizando o conteudo local e mantendo a API
  como apoio.
