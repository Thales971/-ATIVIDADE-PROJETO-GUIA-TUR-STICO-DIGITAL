TUTORIAL DE INSTALACAO E CONFIGURACAO
Projeto: Guia Turistico Digital
Versao: beta de edicao rapida

OBJETIVO
Este arquivo deve ser atualizado sempre que o projeto mudar.
Use como guia unico para os 3 colaboradores.

1) PRE-REQUISITOS
- Node.js 20 ou superior.
- Git.
- Expo Go no celular, se for testar em device.
- VS Code.

2) CLONAR E ABRIR
- git clone <URL_DO_REPOSITORIO>
- cd Guia

3) INSTALACAO
- npm install
- Se o ambiente estiver limpo ou faltando pacotes do Expo, rode:
  npx expo install @react-navigation/native @react-navigation/drawer @react-navigation/bottom-tabs @react-navigation/native-stack react-native-gesture-handler react-native-reanimated react-native-screens react-native-safe-area-context react-native-paper react-native-vector-icons react-dom react-native-web babel-preset-expo

4) EXECUCAO
- npx expo start
- npx expo start -c para limpar cache
- npx expo start --web para validar no navegador

5) ESTRUTURA REAL DO PROJETO
- App.js
- src/screens/WelcomeScreen.js
- src/screens/HomeScreen.js
- src/screens/DetailsScreen.js
- src/screens/AboutScreen.js
- src/screens/ContactScreen.js
- data/places.json
- data/placesRepository.js
- babel.config.js
- app.json

Obs: se existir uma pasta raiz screens/, ela nao e a estrutura principal agora. O codigo real fica em src/screens/.

6) O QUE EDITAR EM CADA CASO
- Navegacao, tema, Drawer, Tabs e Stack: App.js
- Tela inicial e aceite dos termos: src/screens/WelcomeScreen.js
- Lista de pontos e restaurantes: src/screens/HomeScreen.js
- Tela de detalhes com route.params: src/screens/DetailsScreen.js
- Paginas institucionais: src/screens/AboutScreen.js e src/screens/ContactScreen.js
- Conteudo dos cards: data/places.json
- Troca futura para API: data/placesRepository.js

7) FLUXO DE ATUALIZACAO
- Mudou texto: atualize data/places.json ou este tutorial.
- Mudou visual: ajuste App.js e as telas em src/screens.
- Mudou dependencias: atualize package.json e rode npm install.
- Mudou a estrutura: atualize README.md e este tutorial.
- Mudou a origem dos dados: ajuste apenas data/placesRepository.js.

8) CHECKLIST ANTES DE ENVIAR
- O app abre sem erros.
- A tela Welcome abre antes das Tabs.
- O botao Seguir em frente leva para Tabs.
- Drawer abre em Início, Sobre e Contato.
- Lista carrega os dados do JSON.
- Detalhes recebe place via route.params.
- O arquivo JSON continua valido.
- O build web funciona quando necessario.

9) PADRAO PARA FUTURAS MUDANCAS
- Mantenha os textos curtos ate o Figma fechar.
- Evite criar novas camadas de pasta sem necessidade.
- Use este arquivo como fonte principal para os colaboradores.
- Sempre que uma mudanca estrutural acontecer, atualize este tutorial no mesmo PR.

FIM
